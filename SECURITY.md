# Security policy

## Supported versions

Until the project reaches 1.0, only the latest tagged release receives security fixes.

## Reporting a vulnerability

Do not open a public issue for a vulnerability. Use GitHub's private vulnerability reporting for this repository instead. Include affected versions, reproduction steps, impact, and any suggested mitigation.

Notification text is sensitive user data. Reports involving unintended network access, shell execution, path handling, model loading, or leakage of notification content are especially important.

The project does not provide a remote service and must not transmit notification text to external APIs.
